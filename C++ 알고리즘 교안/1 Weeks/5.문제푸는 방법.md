# 문제푸는 방법
1. 문제 해석
    - 문제에서 요구하는 바를 도식화 할 수 있어야함
    - ex) 백준 바이러스 문제 -> (벽 3개 세우기 - 효율? 무식하게?) -> (바이러스 퍼뜨리기 - BFS? DFS?) -> (안전영역 갯수 구하기)
2. 손코딩
    - 내가 알아볼 수 있을 정도로 짜기

- 문제푸는 플로우
    1. 문제의 최대, 최소 범위 파악
    2. 단순 구현은 구현
    3. 최대범위 기반, 시간복잡도상 무식하게 풀 수 있다면 무식하게 모든 경우의 수 생각하면서 풀기 (bruteforce)
    3. 다른 알고리즘 생각해보기(DP, 그리디 등등)
    4. 제출전 반례 항상 생각(외부 입력과 내부 입력은 항상 다르기 때문에 내부 입력 생각해보기)
