# 비트마스킹
-   정의
    - 특정 원소를 찾을 때 시간복잡도
        - 배열: O(N)
        - sorted array: 이분 탐색 O(logN)
        - 해싱 테이블: O(1)
    - bool 배열을 통해 find 메서드를 사용하는 등 연산을 통해 무언갈 찾겠지만 비트 마스킹을 사용하면 더 빠르고 가볍게 사용 가능
    - bool 배열 역할을 하는 "하나의 숫자"를 만들어 "비트 연산자"를 통해 탐색, 수정 등의 작업을 하는것 
- 왜 비트마스킹이라하나?
    - 컴퓨터의 메모리는 0,1이 저장
    - 이때 메모리의 용량의 최소 단위를 bit라 함
    - 이를 기반으로 해당 비트를 키거나 끄너나 하기 때문에 비트마스킹이라 함

- 최종 팁
    - combi(조합)에 활용하면 combi함수를 만들지 않고 for문 하나로 처리가 가능하다
    - 비트 마스킹은 int의 한계까지 가능하므로 31까지만 된다
    - long, long long같은 자료형은 시간복잡도상 불가능(int만해도 10억가지 표현이 되기 때문)
    - 즉 조합, 최대 범위가 31이하면 활용 가능
